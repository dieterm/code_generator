Implementeer in project CodeGenerator.Core.Workspaces.Datasources.OpenApi de ondersteuning voor een openapi/swagger (*.yaml of .json) datasource.
Voeg indien nodig een nuget-package toe om de yaml of json schema te kunnen inlezen.
Probeer het op een gelijkaardige manier te implementeren als een Json datasource.
Het is enkel de bedoeling om in een openapi-bestand de 'components.schemas' sectie te scannen en de objecten als TableArtifact te kunnen importeren.
Wanneer een Property als datatype verwijst naar een ander object (bv. $ref: '#/components/schemas/Category') mag je als generictype 'json' gebruiken, en moet je onder die ColumnArtifact opnieuw een TableArtifact toevoegen voor dat datatype (recursief).
Je kan het project CodeGenerator.Core.Workspaces.Datasources.Json als voorbeeld nemen voor een Json datasource.

Maak ook een OpenApiDatasourceController aan.
Je kan de gelijkaardige controller voor Json als voorbeeld vinden in D:\Cloud\GitHub\code_generator\src\CodeGenerator.Application\Controllers\Workspace\JsonDatasourceController.cs.
Voeg de OpenApiDatasourceController in functie WorkspaceTeeViewControler.LoadArtifactControllers() toe. 
En ook in de dependency injection configuratie: D:\Cloud\GitHub\code_generator\src\CodeGenerator.Application\ServiceCollectionExtensions.cs.

Genereer in project "CodeGenerator.Presentation.WinForms" ook een OpenApiDatasourceEditView gelijkaardig aan JsonDatasourceEditView 
Voeg die nieuwe OpenApiDatasourceEditView daarna toe in functie WorkspaceDetailsView.ShowDetailsControl()
Gebruik een CodeGenerator.UserControls.Views.FileField control om een .yaml of .json-file te selecteren.

## Quick Reference

### MVC (Model-View-Controller)
- **View** ? observeert Model, stuurt user input naar Controller
- **Controller** ? verwerkt input, update Model
- **Model** ? bevat data en business logic
- **Gebruik**: ASP.NET Core MVC, Spring MVC, Ruby on Rails

### MVP (Model-View-Presenter)
- **View** ? passief, delegeert alles naar Presenter
- **Presenter** ? bevat presentation logic, update View expliciet
- **Model** ? bevat data en business logic
- **Gebruik**: WinForms, Android (traditioneel), Web Forms

### MVVM (Model-View-ViewModel)
- **View** ? bindt aan ViewModel properties
- **ViewModel** ? exposed bindable properties en commands
- **Model** ? bevat data en business logic
- **Gebruik**: WPF, MAUI, Blazor, Angular, Vue.js

### Microkernel (Plug-in Architecture)
- **Core** ? minimale kern met basis functionaliteit
- **Plug-ins** ? uitbreidingen die contracts implementeren
- **Registry** ? beheert beschikbare plug-ins
- **Gebruik**: IDE's (VS Code), Editors, Code Generators, CMS

## Gemeenschappelijke Principes

Alle patterns delen dezelfde kernprincipes:

1. **Separation of Concerns**: UI logica gescheiden van business logic
2. **Testbaarheid**: Business logic testbaar zonder UI
3. **Onderhoudbaarheid**: Wijzigingen in één laag beïnvloeden andere lagen minimaal
4. **Single Responsibility**: Elke component heeft één verantwoordelijkheid

## Relatie met Design Patterns

Architectural patterns gebruiken vaak Design Patterns intern:

| Architectural Pattern | Gebruikte Design Patterns |
|----------------------|---------------------------|
| MVC | Observer, Strategy, Composite |
| MVP | Observer, Mediator |
| MVVM | Observer, Command, Mediator |
| Microkernel | Strategy, Factory, Registry |
